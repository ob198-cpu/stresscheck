# ストレスチェック受検システム MVP

外部実施者として委託を受け、従業員がスマホブラウザで回答し、個人結果を会社から隔離するための最小システムです。

## 起動

```powershell
node server.js
```

ブラウザで以下を開きます。

```text
http://localhost:8787
```

## 画面

| 画面 | 内容 |
|---|---|
| 従業員回答画面 | 本人専用URLから回答 |
| 本人結果画面 | 回答後に本人へ直接結果通知 |
| 会社担当者画面 | 受検済み/未受検、10人以上の集団分析のみ表示 |
| 実施者画面 | 個人結果、高ストレス判定、同意、面接申出を確認 |
| QRコード付き受検案内 | 受検者ごとの本人専用URLとQRコードを印刷 |
| 社内メール文 | 本人専用URLを社内メールで配布する文面を表示 |

## 権限設計

会社担当者画面には以下を表示しません。

- 回答内容
- 個人結果
- 個人別スコア
- 高ストレス者判定
- 面接申出前の高ストレス者氏名
- 10人未満の部署別分析

## 受検者への配布

実施者画面から `QRコード付き受検案内を作成` を開きます。

受検者ごとに以下が表示されます。

- 氏名
- 所属
- 本人専用URL
- QRコード

印刷して本人へ配布するか、`社内メール文を見る` から本人専用URLをメール配布します。

QRコードはローカルで生成します。外部のQR生成サービスへ本人専用URLを送信しません。

## 本番化で必須の対応

このMVPは制度設計と画面分離の試作品です。本番運用前に以下を必ず実施してください。

- HTTPS環境で公開する
- `PRACTITIONER_KEY` を強固な値に変更する
- 管理者ログインをURLキー方式から正式な認証に変更する
- データベースを暗号化・バックアップする
- アクセスログ、操作ログ、削除ログを強化する
- 設問と判定基準を厚労省推奨方式で最終確認する
- 契約書、個人情報条項、実施規程を弁護士・社労士等に確認してもらう

セキュリティ上の注意は [SECURITY.md](SECURITY.md) を確認してください。

## 重要

Google Forms のような通常アンケートではなく、外部実施者が個人結果を管理し、会社に個人結果を見せないための専用システムとして扱います。
