Web system / stress check / workplace support

毎月の面倒な業務を、判断しやすい仕組みに変える。

InfoMは、紙・Excel・メールで回している業務を整理し、必要なところだけWebシステム化します。 ストレスチェックや講義も、実施して終わりではなく、受検状況、本人通知、面接指導申出の案内まで業務として回る形にします。

Why it feels different

画面を作る前に、業務の詰まりを特定します。

「フォームを作りたい」と言われても、実際の問題は入力後の確認、催促、集計、報告、権限にあることが多いです。 InfoMでは、作る前に業務の流れを見て、改善効果が出る場所から着手します。

入力誰が、いつ、何を入れるかを整理
確認担当者、期限、未対応を見える化
共有会社に見せる情報と見せない情報を分ける
報告毎回作り直さず、出力できる形に残す

Data Trust Layer

本人に返す情報と、会社が見る情報を分けて設計します。

ストレスチェックでは、見た目よりも情報の分け方が信頼に直結します。 本人結果、受検状況、集団分析、面接指導申出を混ぜずに扱い、担当者が迷わない運用導線を作ります。

本人結果は本人向けに整理 会社向けは集計・進捗を中心に表示 権限と出力範囲を事前に設計

Project Handoff

最初の相談で、作るものと作らないものを切り分けます。

いきなり大きなシステムを作るのではなく、今の資料を見て、手作業で残す部分とWeb化する部分を分けます。 そのうえで、回答回収、確認、通知、報告出力までを必要なサイズで納品します。

既存資料を見ながら範囲を決める 初回から運用担当者が使う画面を想定する 納品後の修正・更新方法も整理する
